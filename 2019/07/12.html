Problem (and here is where I teach computer)

The annoying part right now is that my link doesn't work while I am developing the website on my computer. For those who aren't familiar with web development, there are different "environments" in which a website can run.

Production

What you typically see is called the production environment. This is when the website is completely visible to the public. For this site, kylethornburgh.com is the production environment. I can view it on my phone, send a link to my friends, Google can find it, whatever. No restrictions (unless I place restrictions on it). However, this doesn't require some magical sacrifice to the internet gods to make my site visible to the world. It's just running on a computer. I could run a publicly visible website on my own computer if I wanted to. But then if I ever needed to update my computer and restart it, my website would go down. This is why websites are generally being run on someone else's computer, AKA "the cloud." Amazon is much better at keeping their machines running than I am. And if their computers need to do a software update? They have other computers that will keep your website running while they handle the update. This is why so many people use Amazon, or any other hosting providor, instead of running their own servers.

Development

Then there is the development environment. This is typically done on the developer's own machine. It's just like if I were to use my computer to run my site for the whole world to see, but instead it's set up in a way that makes the website only visible to me. Now, instead of going to kylethornburgh.com, I type "localhost" into my web browser. This is telling my browser, "Hey instead of searching the internet for this website, just look for a server running on this computer."

That last bit is what is causing the issue with my link that I am adding. I can't just link to kylethornburgh.com because that will take me from my development environment to my production environment. But I want to stay in my development environment while I am trying to test my site. I need to tell my computer to go back to the homepage of my localhost.

Staging

Another popular environment is the staging environment. This is very similar to the production environment, but now with some added restrictions. In a staging environment, you generally want to limit who can see your website. This is done to allow certain people to test your website in an environment that is as close as possible to your production environment. However, you don't want just anyone to find this version because it is much more likely to be broken.

You can do a couple things to place restrictions on this staging environment. You can make the website require a password. This is useful if you want to give anyone the ability to view the staging website, but not without your permission. Generally this approach is better if you are building a website for clients who want to be able to view the staging site. You can also require that the staging site is viewed while connected to a specific wifi. That's probably better for development teams without external clients because you don't have to enter a password each time you want to test the site. But you do lose the ability to share the website with other people.
